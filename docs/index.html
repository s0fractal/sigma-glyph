Σ-GLYPH

Deterministic compute by content. The same term with the same budget yields the same result hash — on any machine, across conforming implementations.

A content-addressed SKI evaluation core with a bit-exact determinism guarantee. Integer-only, total, budgeted. v0.6.6.

result_hash = eval(term_hash, atp)   // deterministic · integer-only · total

Compute one thing

eval is total: every term with every budget resolves to exactly one canonical outcome — a normal form, ATP Exhausted, or Unresolved Reference. A non-terminating term is not an error — it is a priced, canonical outcome. (A concrete implementation may additionally hit a local limit, e.g. stack depth, and raise a non-canonical resource fault; by design those are never part of the canonical result.) Work and peak materialised size are bounded by the budget (size − 1 ≤ spent, a proved invariant — see below).

$ python3 <<'PY'
# I K  ->  K      (a two-step reduction)
# then run Omega = (S I I)(S I I) forever, with a budget of 1000
PY
result: bc0c2fe26e44e2ae spent: 4
Omega:  ATP Exhausted    spent: 975

Two conforming implementations, same spec version and same inputs, get byte-identical result hashes. That is the whole of Book I. Full runnable snippet in QUICKSTART.md.

What it is / what it is not

It is

  • A total, deterministic evaluator whose result's identity is the SHA-256 of its canonical bytes.
  • A cross-implementation contract: agreement is defined by shared vectors, not by one binary.
  • A priced machine — compute and memory both paid from one budget.

It is not

  • A blockchain, a token, or a consensus network.
  • A general VM or a language runtime — the core is SKI, deliberately tiny.
  • Fast. It trades speed for an exact, portable identity of computation.

Why not WASM / Nix / IPFS / zkVM / just Python?

The fair skeptic's question. Honest answers, each about what Σ-GLYPH builds in:

vs WASMWASM executes portably, but its core format gives a computation no content-derived identity — two toolchains, two byte streams, no shared name for "the same result." Σ-GLYPH's identity is the hash of canonical bytes.
vs NixNix reproduces builds by hashing inputs. It pins an environment, not the semantics of a total, priced evaluation. Different layer — Σ-GLYPH is what runs, not how it was assembled.
vs IPFSThe IPFS protocol content-addresses data and does not compute. Σ-GLYPH content-addresses computation — terms and their results — with a totalization guarantee the storage layer makes no claim about.
vs zkVMA zkVM proves a computation satisfied its relation without re-running it — powerful, and heavy. Σ-GLYPH targets the case where you can re-run: verification by deterministic re-execution. Complementary, not competing.
vs just PythonThe Python impl is only the oracle. The guarantee is the spec + vectors, replayed byte-exact by independent Book I evaluators (a dependency-free Rust one in this repo; warrant's ski@v1 in another). "Just Python" gives one binary's behaviour; this gives a contract between strangers.

Held to its own standard: adversarial review

No change lands on the maintainer's say-so alone. Each substantive change is attacked by independent models before it is adopted, and the critiques and the point-by-point responses are published. Seven distinct model families have reviewed this repository — Claude, Codex, DeepSeek, Gemini, GPT, Kimi, Qwen; the open reviews/ archive collects the external critiques, maintainer responses, and gate records.

Why a panel and not one reviewer: in v0.6.4, one reviewer of three (Gemini) caught a defect in the governance verifier — it refused on the mere presence of a settled key-state warrant, so the first roster rotation would have frozen the chain permanently. A single sign-off would have shipped it. It is fixed, with a regression vector, and recorded in the CHANGELOG.

Governed by its own constitution

The specification amends itself through a written process, not a maintainer's fiat. Releases are anchor-set blobs adopted by 2-of-3 threshold warrants — Ed25519-signed, hash-addressed decision records (see the sister project warrant). The constitution — GOV-anchors.md — is deliberately not one of the Books: a document must not judge itself.

Machine-checked, and where the trust actually sits

  • Determinism and totality of the Lean evaluator are definitional (well-founded step, fuel-indexed total eval; no partial/unsafe). The memory bound size − 1 ≤ spent is a theorem on the concrete evaluator (evalHash_size_bound), compared with the Python oracle on 33 Book I vectors.
  • Three independent Book I evaluators — the Python oracle, this repo's dependency-free Rust (its own SHA-256, no external crates), and warrant's ski@v1 — agree on every Book I vector.
  • TCB honesty: SHA-256 is a total Lean function, but the genesis pins H(I)/H(K)/H(S), the SHA test vectors and the Book II LUT facts are discharged with native_decide — so those checks trust the Lean compiler, not only the kernel. Only the pure integer lemmas are kernel-only. The full breakdown is in proofs/README.

Scope & non-goals

Vector agreement is evidence of conformance at the tested points; the Lean theorems cover all inputs only for the properties they state (totality, determinism, the size bound). Σ-GLYPH makes no cryptographic claim beyond SHA-256's, and none about sandboxing, side channels, or adversarial resource isolation of a host. It is a definition of what computes to what, not a secure execution environment.
