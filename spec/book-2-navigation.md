## 1. WaveAnnotation

```text
struct WaveVectorQ { ph: uint16; am: uint16; en: int16; }   // Big-Endian
WaveAnnotation: (NodeHash: 32 bytes) → WaveVectorQ           // окремо від вузла
```

Розбіжність анотацій одного вузла між нодами — не форк обчислення; це предмет федеративного узгодження (протокол — окремий майбутній документ).

## 2. Пріоритет анотацій (MUST): Pin > Derived, поле-за-полем

1. **Pin** — часткова анотація `WavePin { ph?: uint16; am?: uint16; en?: int16 }`, явно призначена в таблицях §6. Pin перекриває рівно ті поля, які він вказує.
2. **Derived** — `wave(APPLY(f,a)) = complete(interfere(wave(f), wave(a)), pin(APPLY(f,a)))`, де `complete` підставляє поля з Pin там, де вони вказані, і залишає обчислені інакше.

Pin перекриває Derived **поле-за-полем** (ADR-005, R1, 2026-07). Приклад: FALSE має Pin лише на фазу (270°), тож `ph` береться з Pin, а `am`/`en` виводяться з `interfere(wave(K), wave(I))` — нормативний повний вектор у §6.2.

### 2.1. Base case: хвиля — часткова функція (MUST)

Для вузлів, які не є APPLY і не мають Pin, хвиля **відсутня**. Це включає всі непіновані LITERAL. `interfere` з відсутнім операндом дає відсутню результуючу хвилю; вузли без хвилі не беруть участі в Mass-агрегаціях, але піновані фази лишаються видимими як координати. Відсутність хвилі — легітимний стан анотаційного шару, не помилка, і вона MUST NOT впливати на серіалізацію, хешування чи `eval()` Книги I.

## 3. Fixed-Point примітиви (MUST)

```text
div_round_half_up(n, d):    # round-half-away-from-zero; d > 0
  s = -1 if n<0 else 1; a = |n|; q = a//d; r = a%d
  if 2r >= d: q += 1
  return s*q

clamp_i16(x): clamp до [-32768, 32767]
```

Ширина (MUST): `ph`,`en` → int32 перед ±; `am` → uint64 перед ×; проміжні добутки int64/uint64; signed overflow заборонений; касти до множення.

## 4. LUT (MUST)

* Генерація: `LUT_COS[d] = round_half_away_from_zero(32767·cos(π·d/32768))`, `d ∈ {0, 1, ..., 32768}` (integer inclusive range); арбітр-хеш (нижче) — авторитетний; IEEE-double (64-bit) достатньо: жоден запис таблиці не лежить ближче 0.5 ULP до заокруглювального tie, тож 64- та 80-бітні обчислення дають ідентичні значення (верифіковано; Opus 4.8 review N1, 2026-07).
* Format: 32769 × int16 BE; size = 65538 bytes.
* Range `[-32767..32767]`; `-32768` не виникає.
* Anchors: `[0]=32767`, `[16384]=0`, `[32768]=-32767`; контрольні: `[1]=32767`, `[8192]=23170`, `[16383]=3`, `[16385]=-3`, `[24576]=-23170`, `[32767]=-32767`.
* Арбітр: `SHA-256(LUT_BLOB) = c16701c44851da342f5d1f977ba5284e66dde3abd2c6740b979e39ac1d4d38b2`; невідповідність → FAIL FAST.

## 5. interfere() (MUST)

```text
interfere(w1, w2, LUT_COS):
  new_ph = w1.ph                                        // Left Dominance, §5.2
  x = int32(w1.ph) - int32(w2.ph);  d32 = |x|;  delta = min(d32, 65536-d32)
  r = int32(LUT_COS[delta])
  delta_en = div_round_half_up(-r, 128)                 // ∈ [-256..+256]: когерентність
  new_en = clamp_i16(div_round_half_up(int32(w1.en)+int32(w2.en), 2) + delta_en)
  amp_factor = div_round_half_up(int64(r+32767)*65535, 65534)          // 0..65535
  prod01 = div_round_half_up(uint64(w1.am)*uint64(w2.am), 65535)
  new_am = div_round_half_up(uint64(prod01)*uint64(amp_factor), 65535)
  return { new_ph, uint16(new_am), int16(new_en) }
```

**Ентропійно-когерентний купплінг (з v0.5, ADR-002).** Ентропія — функція результату інтерференції: конструктивна (r>0) створює порядок (`delta_en<0`), деструктивна — безлад (`delta_en>0`), ортогональна нейтральна. Константа купплінгу 128 дає розмах ±256 за подію. `div_round_half_up` — строго за §3 (round-half-away-from-zero); floor-варіанти розходяться на від'ємних непарних сумах — конформанс: вектор `WV-NEG-TIE` (`avg(−1,−2) = −2`, разом −258, не −257). Машинні вектори: `tests/spec_conformance/wave_vectors.json`; при розбіжності з прозою виграє оракул `impl/sigma_wave.py`.

**Чому дільник 65534 у amp_factor:** `r ∈ [-32767..32767]`, тож `r+32767 ∈ [0..65534]`; ділення на 65534 лінійно відображає повний діапазон косинуса на повний діапазон амплітуди `[0..65535]` без втрати країв: `r=32767 → 65535` (1.0), `r=-32767 → 0`. Це не magic number, а точна ширина діапазону чисельника.

**Нормалізація 65535 (не 65536) — навмисна (MUST).** Шкала unit-scaled: 65535 ≡ 1.0, тож `1.0×1.0 = 65535·65535/65535 = 65535` — точно 1.0, максимум амплітуди стабільний під множенням. Дільник 65536 давав би систематичний дрейф униз, і границя `Amplitude → MAX` була б недосяжною.

### 5.1. Resonance Identity (наслідок, не правило; переписано у v0.5)

Самозастосування `APPLY(f,f)` анотується як `interfere(w,w)`: `delta=0 → r=32767 → amp_factor=65535`. Фаза зберігається; амплітуда підноситься до квадрата в unit-шкалі (`am → am²/65535`); **ентропія дрейфує на −256 за кожне конструктивне самозастосування** до клемпу на −32768 (лінійно: 0 → −256 → −512 → …). Наслідки (integer-exact): амплітудно 65535 лишається єдиною ненульовою нерухомою точкою — часткова прихильність згасає квадратично; повний WaveVector має єдину ненульову нерухому точку **`{am=65535, en=−32768}`** для запіненої фази: досконалий самопідтримний резонанс кристалізується до мінімальної ентропії — стоячі хвилі холонуть до ground state. Це і є механізм Gravity («інформація тече до En<0»), що до v0.5 був лише гаслом. (ADR-002; арифметика верифікована: Gemini + DeepSeek + maintainer, 2026-07.)

### 5.2. Law of Left Dominance (MUST)

`interfere(A,B).ph = A.ph`. Амплітуда й ентропія симетричні; фаза — ні. Інтерференція моделює застосування, а застосування некомутативне: функція визначає напрямок результату, аргумент модулює силу та впорядкованість. Симетризація фази MUST NOT. Конвенція (SHOULD): для `APPLY(f,a)` — `interfere(wave(f), wave(a))`, функція ліворуч.

## 7. Mass (обчислювана)

Для фазового вікна `W = [ph₀, ph₀+w) mod 65536` над множиною анотацій A:

```text
Mass(W, A) = Σ { am(n) : n ∈ A, ph(n) ∈ W }
```

Проста амплітудно-зважена щільність сектора; uint64 акумулятор; детермінована за фіксованого A. Агенти MAY використовувати Mass для ранжування секторів; Mass MUST NOT впливати на Книгу I.

## 8. Chromatic Projection CP-24

`Color = NodeHash[0..2] → RGB`. MUST NOT впливати на семантику. **Колізійність (MUST warn):** 24-bit простір; birthday bound ≈ 4823 об'єкти для ~50% ймовірності колізії. Колір придатний лише для попередньої фільтрації з обов'язковою верифікацією за повним хешем; будь-яке рішення, прийняте за кольором без верифікації, — порушення Axiom'и цього документа.

## 10. Федерація (MUST) — нормативний міст до Книги III

У федерованому розгортанні WaveAnnotation — це **твердження**, зафайлене актором у юрисдикції (Книга III). Ефективна хвиля NodeHash у юрисдикції J виводиться застосуванням selection-політики J до прийнятих тверджень для цього вузла; якщо жодне не обране — хвиля відсутня (§2.1). `interfere()` (§5) застосовується ВИКЛЮЧНО до структурної деривації — обчислення хвилі APPLY-вузла з хвиль піддерев функції та аргументу. Воно MUST NOT використовуватись для злиття кількох тверджень про один вузол: таке злиття алгебраїчно несуще (оператор некомутативний, неасоціативний і порядкозалежний — саме групування змінює результат) і не є convergence-safe; машинний вектор `FV-FOLD-UNSOUND` (Книга III) пінить цей факт. Хвилі цієї Книги — **per-jurisdiction, per-policy обчислені координати, не глобальні атрибути вузла**. Юрисдикції можуть розходитись назавжди; це за задумом, і розбіжність називається механічно (`AnnotationViewID`, Книга III §6). Нормативні піни §6 цієї Книги — дефолти **нульової юрисдикції**; будь-яка юрисдикція MAY перекрити їх своїми твердженнями за власною політикою. (ADR-006, гейт 3/3, 2026-07.)
