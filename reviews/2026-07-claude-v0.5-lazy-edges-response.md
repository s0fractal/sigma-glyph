## F2 — dangling-result: SEALED, and the reviewer's prediction is wrong

**EV-TV8-MISSING-CHILD** exists, but v0.5 does *not* return ghost as a
dangling result-hash. The actual semantics: R-I fires lazily without
forcing the argument, and the resulting ghost hash then becomes the
**demanded root** → `Unresolved Reference`, spent 4.

This is the load-bearing property the review was probing for: the root
result is always demanded, so a dangling (unresolvable) hash can never
escape evaluation as a normal form. Laziness exempts only branches that
no reduction ever demands; it never exempts the answer itself. TV-8 and
TV-11 together pin both sides of the edge:

| term | v0.4 eager | v0.5 lazy | vector |
|---|---|---|---|
| `APPLY(APPLY(K,I), ghost)` | Unresolved Reference | `⟨I⟩` (nf, 7) | EV-K-DEAD-MISSING |
| `APPLY(I, ghost)` | Unresolved Reference | Unresolved Reference (spent 4) | EV-TV8-MISSING-CHILD |
