## P1 findings

### P1: Python `eval_hash` can return a term beyond its configured resource limits

Evidence: `impl/sigma_glyph.py` checks materialized size and depth only when
`steps % 256 == 0`:

```python
steps += 1
if steps % 256 == 0:
    if size(t) > limits["max_materialized_nodes"]:
        raise ResourceFault("term growth")
    if depth(t) > limits["max_node_depth"]:
        raise ResourceFault("term depth")
```

If evaluation reaches normal form before step 256, line 213 returns the term
without a final limit check. The following probe uses the canonical `FALSE`
APPLY node and reproduces both violations:

```bash
python3 - <<'PY'
import sys
sys.path.insert(0, "impl")
import sigma_glyph as s

st = s.Store()
root = st.put(s.FALSE_BYTES)

for depth_limit in (1, 2):
    try:
        term, spent = s.eval_hash(
            root, 10, st,
            dict(max_node_depth=depth_limit,
                 max_materialized_nodes=100,
                 max_store_fetches=100))
        print("depth_limit", depth_limit, "RETURN", s.depth(term), spent)
    except Exception as exc:
        print("depth_limit", depth_limit, type(exc).__name__, exc)

try:
    term, spent = s.eval_hash(
        root, 10, st,
        dict(max_node_depth=100,
             max_materialized_nodes=1,
             max_store_fetches=100))
    print("size_limit 1 RETURN", s.size(term), spent)
except Exception as exc:
    print("size_limit 1", type(exc).__name__, exc)
PY
```

Observed:

```text
depth_limit 1 RETURN 2 3
depth_limit 2 RETURN 2 3
size_limit 1 RETURN 3 3
```

The depth-limit-2 return is valid; it is included as a control. The
depth-limit-1 and size-limit-1 returns exceed their configured limits.
This contradicts the function's explicit contract:

```text
Resource limit breach -> ResourceFault (local, non-canonical).
```

This is not a request to make resource faults canonical. Sampling cadence may
remain an implementation choice, but a setting named `max_*` must not allow a
completed return beyond that maximum.

Concrete fix:

1. Factor the size/depth test into a local helper.
2. Keep the 256-step sampling if desired for long-running evaluations.
3. Run the helper after every successful materializing/rewrite step that may
   increase size or depth, or at minimum immediately before every normal-form
   return.
4. Preserve the existing Omega behavior: high ATP with a tiny cyclic term must
   still yield canonical ATP Exhausted rather than a size fault.

Required regression cases:

- `FALSE`, `max_node_depth=1` raises `ResourceFault("term depth")`.
- `FALSE`, `max_materialized_nodes=1` raises
  `ResourceFault("term growth")`.
- Exact-bound controls (`depth=2`, `size=3`) return normally.
- Existing depth-1500 and Omega tests remain unchanged.
- If Rust exposes equivalent configurable limits, either pin parity or
  explicitly document that these controls are Python-local.

## P2 findings

### P2: the repository has no single root command for the complete validation matrix

`go test ./...` from the repository root fails because the Go module lives in
`impl-go/`; `cargo test` succeeds while running zero Rust unit tests because
the real Rust gate is the custom `conformance` command. The CI is correct, but
the obvious local commands give incomplete or misleading results.

Concrete fix:

Add a root `Makefile`, `justfile`, or checked-in script with a `test-all`
target that runs the same Python, Go, Rust, differential, anchor, and bridge
commands as CI. CI should invoke that target or verify it stays equivalent.

This is P2 only: the current CI commands themselves passed.
